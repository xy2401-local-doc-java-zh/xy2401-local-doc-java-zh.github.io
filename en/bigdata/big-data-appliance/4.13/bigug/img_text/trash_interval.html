Description of the illustration trash_interval.png

This screen capture of the HDFS property settings in Cloudera Manager shows the following elements, listed from top to bottom and from left to right:

hdfs, green circle for Good Health, Actions menu.

Tabs: Status, Instances, Commands, Configuration (selected), Audits, Charts Library, File Browser, Replication. WebUI menu.

Search field with "trash" entered, Remove Filter button, text field, Save Changes button.

The main panel displays the properties. The following properties and values are shown in a tabular format:

The header row of the main panel displays Categoy, Property, Value, and Description column headers.

Category column displays Gateway (Base) and NameNode Default Group

Property column displays Use Trash and Filesystem Trash Interval fs.trash.interval

Value column displays Selected options box Reset to the default value: false HDFS Trash is enabled and 1 day(s) default value Trash checkpointing is on

Description column displays Move deleted files to the trash so that they can be recovered if necessary... and Number of minutes between trash checkpoints... To disable the trash feature, enter 0.
